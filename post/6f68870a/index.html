Coding Pages申请SSL证书失败踩坑

发表于 2019-11-03 更新于 2020-02-16 分类于 技术 ， 经验踩坑 阅读次数： Valine：

起因：Hexo博客双线部署

由于国内访问 Github 有很高延迟，且 Github Pages 屏蔽百度爬取。因此我把 Hexo 同时部署到 Coding 和 Github，在云服务器的 DNS 设置分区解析，国内线路解析到 Coding，国外线路解析到 Github（虽然国外没人看就是了）。


1. 问题

在配置 DNS 后访问站点出现 HSTS异常 。原因是 Coding 不像 Github 默认就是 https 协议，但Coding 也在设置里提供了免费的 SSL证书 申请（Let’s Encrypt），那就用它吧。


2. 踩坑

事不宜迟开始申请。快速地踩了两个坑（PS: 半小时才能申请1次是真的烦）：

问题原因： 验证域名所有权时定位到 Github Pages 的服务器导致 SSL 证书申请失败


问题原因： 海外线路没有解析到 Coding Pages 的服务器。


总的来说，SSL证书 是通过 Let’s Encrypt API 申请。申请证书前需要验证域名，而 Let’s Encrypt 位于国外，所以需要保证 Let’s Encrypt 能通过您的域名正常访问到 Coding Pages 服务器以读取验证信息。


3. 解决方法

先把分区域名解析停用，配置默认线路的解析到 xxxxx.coding.me，完成申请SSL证书后启用原来的解析。如果原先博客用着Coding的，在双线部署前申请就没有上述问题了。

其它情况参考官方帮助文档 => Coding Pages 常见问题，帮助文档总结的很全，要养成看文档的习惯。

但是还有一个很严重的问题就是申请到的SSL证书仅有4个月使用期，过期后要重新申请，步骤也是如上所述，非常繁琐。

说实话Coding现在不是很稳定，而且今年一月份暂停服务升级，导致我的博客上不去（文章修改于2020/2/11）。所以接下来我应该会把博客迁回我自己的服务器。
